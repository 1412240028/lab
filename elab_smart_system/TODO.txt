🗂️ Refactor Planning — elab-ui.css
✅ Fase 0 — Persiapan (sebelum mulai)
Jangan langsung hapus/edit file lama dulu.

 Backup elab-ui.css → rename jadi elab-ui.OLD.css, simpen di folder yang sama
 Buat struktur folder css/ seperti saran sebelumnya
 Buat file main.css kosong sebagai entry point
 Di semua file HTML/PHP, ganti <link href="elab-ui.css"> jadi <link href="css/main.css">
 Test dulu — pastiin tampilannya masih oke (karena main.css masih kosong, tampilan bakal rusak sementara, itu normal)


🔧 Fase 1 — Base & Global (mulai dari sini)
Ini fondasi, harus selesai duluan sebelum lanjut ke fase lain.
Target file:

css/base/variables.css → cut semua isi blok :root { } dari file lama
css/base/global.css → cut * { } dan body { } dari file lama

Di main.css tambahkan:
css@import "base/variables.css";
@import "base/global.css";
Test di browser → background dan font harus muncul normal.

💡 Tips selama refactor
Cara kerja yang aman: gunakan cut (bukan copy) setiap mindahin class. Jadi kalau ada yang ketinggalan, masih keliatan di file lama.
Cara cek apakah ada yang ketinggalan: buka DevTools browser → Inspect element yang tampilannya aneh → lihat di tab Styles, CSS-nya dari file mana. Kalau masih dari elab-ui.OLD.css, berarti belum dipindah.
Jangan refactor sambil nambahin fitur baru — pisahkan dua kegiatan itu biar gak bingung kalau ada bug.
